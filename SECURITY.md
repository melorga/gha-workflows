# Security Policy

## Supported Versions

Only the latest tagged release of this repository receives security fixes. Consumers are encouraged to pin to a tagged release (see the Versioning section in the README) and upgrade promptly when new releases are published.

## Reporting a Vulnerability

If you discover a security vulnerability in these reusable workflows, please report it privately rather than opening a public issue.

- Preferred: open a GitHub Security Advisory via the **Security** tab of this repository ("Report a vulnerability").
- Alternative: contact the maintainer directly through their GitHub profile (@melorga).

Please include:

- A clear description of the issue and the affected workflow / file.
- Reproduction steps or a minimal proof-of-concept.
- Any known impact (e.g. credential exposure, privilege escalation, supply-chain risk).

## Disclosure Process

1. We will acknowledge receipt within **5 business days**.
2. We will investigate and confirm the issue, then work on a fix on a private branch.
3. Once a fix is available, we will publish a new tagged release and a GitHub Security Advisory crediting the reporter (unless anonymity is requested).
4. Please refrain from public disclosure until a fix has been released.

Thank you for helping keep this project and its consumers safe.
